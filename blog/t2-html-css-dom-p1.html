    • Can you think of an analogy to describe HTML, CSS and the DOM? How would you describe it to your non-technical friend?
    I think it will take some more time to really appreciate the differences and interactions between the three but, at present, my understanding is that HTML is the content/raw information you wish to include in your website, and the CSS is the styling or how you want to present things.
    The DOM (Document Object Model) is not as clear to me but I understand there is a recognised standard (using a defined 'tree-model') that browsers use to read and then display HTML and CSS as web pages, I'm still unclear on if the DOM is the standard itself or the process of converting and displaying the HTML+CSS provided into the HTML+CSS output using said standard.
    I think it is the later which, to me, says the DOM is more of a process (more verb than noun) happening between the content+style and the standardised model (by the browser/other) to create a final output.
    I could use an art exhibition as an analogy; The paintings are the HTML, how they are framed up is the CSS, and the curation is the whole process to set these elements up and display them in the given space using the standard gallery fittings/lighting/etc.
    If the DOM is the recognised standard rather than the process then an analogy could be that the DOM is like the instructions in a box of model pieces, the html would be the pieces and any paint/stickers you apply on the pieces is the CSS.


    What is meant by boxifying design?
    Boxifying design means dividing all the HTML elements of your project in boxes.
    The whole webpage is one box, every element within the webpage can be boxed up too e.g. headers, tabs, buttons, blocks of text, etc.
    There can be boxes inside other boxes and boxes overlapping. All boxed elements can be tweaked individually or can be tweaked relative to other boxes.
    This kind of design allows us to perform complex arrangements with basic content. CSS elements can be applied to certain 'boxes' of HTML in isolation or in relation to other 'boxes'.


    what is the box model?
    The box model is a useful (and common) way of interacting with our HTML elements. It includes four parts: content, padding, border and margin.
    Each of these parts relate to each and every box within our box-designed HTML. Using the box model, if we had a 'box' with text based content inside it we can alter/stylise

  • the content - the text in our box
  • the padding - the space between our text and it's boarder
  • the boarder - shape ,form, and whether it is visable or not
  • the margin - the space between this box and another element


    • Utilising 'boxifying design' and the 'box model' enables us to create beautiful designs + styles and play with our web content in some really dynamic ways.
