In the JavaScript language, describe all the use cases for each of:
Parentheses ()
Parentheses let you:
Group or contain expressions or parameters - you do this to make sure what you are trying to communicate is represented in Javascript. Different values/aperators have a different order of precedence and it can be easy to miss the way the expression/parameter we build is being interpreted in JS.
E.g. answer = (x + y) * 2
In this example you want the x and y added FIRST, not the ‘y * 2’ calculated first as would be done otherwise, so you use parentheses around the addition.
If you’re writing an expression and you have spaces in your elements you can chuck this expression into parentheses so it’s received as a single thing.

Display conditions for statements and passing parameters/arguments into a function - If you have a statement within an event/function/loop you chuck it inside parentheses.
This translates to mean if the contents of the parentheses is true do (this), otherwise don’t do (this).
We use an empty parentheses set when we define a function, If a function has no parameters to set we would leave the braces empty. When we call the function we could then put something (parameters) into the parentheses to run it through the function.
E.g. I could have a greetPerson (); function that says “Hello (name)” when called, when I am calling this function I could put Bob’s name into the parentheses to then greet Bob.
So greetPerson (‘Bob’) -> “Hello Bob”

Brackets []
We use brackets to wrap our arrays. Usually for integers and strings, you can put a lot of info into the brackets/arrays, many things. Once in an array we can access any one of those things to use as needed.

Braces {}
We use braces to wrap all the statements within a function. We also wrap nested functions and events (e.g. functions in functions, statement blocks like for {} and if {}).
They can also be used to define initial values in array declarations.

Single quotes '' and Double quotes ""
In Javascript it seems you can use single or double quotes (or backticks ``) for much the same purpose. In general they are all used to contain strings or text that is to be read as text rather than as Javascript. However it is crucial to always close a quote with the same type of quote you opened with. It is generally considered professional etiquette to be consistent with a single pattern in your work, if you or anyone else has used double quotes to wrap text you continue to use double quotes throughout this piece of work.
I have no experience or preference to date on this.

From what I’ve read online more people prefer using single quotes in JS.for internal strings and double for external - you can then distinguish internal constants from strings that are to be displayed to the user.
Fyi Chrome and Firefox’s javascript engines uses double quotations and Node.js uses single.
